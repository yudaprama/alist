## **角色与路径**

- 现在更推荐通过角色来管理用户访问范围。
- 一个角色可以包含多条权限作用域，每条作用域都是 `路径 + 权限位` 的组合。
- 用户的最终权限，是其所有角色作用域的并集。
- `基本路径` 只决定登录后的默认落点，不会替代角色里的权限作用域。

<br/>

## **设备会话**

- AList 通过请求头 `Client-Id`，或 API 请求里的 `client_id` 查询参数识别设备。网页前端会自动生成并持久化一个稳定的 client id。
- `max_devices` 是全局设置，`0` 表示不限制设备数量。
- 当 `max_devices` 大于 `0` 时，`device_evict_policy` 决定超限后的处理方式：
  - `deny`：拒绝新的登录
  - `evict_oldest`：先踢掉最早的活跃设备会话
- `device_session_ttl` 也是全局设置，单位是秒，用来自动清理过期的设备会话记录。
- 这些全局控制项可以在 `后台 => 设置 => 全局` 中配置。
- 用户可以在 `后台 => 会话 => 我的会话` 查看并踢掉自己的设备。
- 管理员可以在 `后台 => 会话 => 管理` 查看全部设备会话。
- 退出登录会把当前设备会话标记为失效。失效后，同一设备会话需要重新登录才能继续使用。

## **Tips**

1. 你是否在为游客能看到全部文件而发愁？[**点击查看这里如何设置**](../../faq/why.md#%E6%83%B3%E8%AE%A9%E6%B8%B8%E5%AE%A2%E7%99%BB%E5%BD%95%E5%90%8E%E6%89%8D%E8%83%BD%E7%9C%8B%E5%88%B0%E5%86%85%E5%AE%B9%E6%80%8E%E4%B9%88%E8%AE%BE%E7%BD%AE)
2. **`Guest user is disabled, login please`**：为了保护您的AList安全，游客访问权限关闭了，若需要游客访问自行打开
   - 后台 --> 用户 --> `guest` --> 停用取消勾选
3. 关于 **`复制/上传`** 的问题
   - 如果你上传(含离线下载上传)很大的文件，或者上传文件很多（几百上千），不建议您使用AList来进行操作,请前往相应的网盘官网直接操作最后
4. 非admin权限用户在后台管理离线下载、复制、上传等操作
   - 需要AList升级到<Badge text="v3.39.1" type="info" vertical="middle" />版本，同时admin也可以查看用户的任务进度以及操作
